# Лабораторная работа №2: Виртуальный датчик для контроля процесса обжига в печи

## Описание

Ноутбук `lab.ipynb` реализует виртуальный датчик — регрессионную модель, которая по текущим и историческим показаниям 16 датчиков телеметрии предсказывает целевой показатель в реальном времени и приближает редкие лабораторные замеры.

Решение включает:
- загрузку и проверку данных;
- EDA по телеметрии и целевой переменной;
- подбор задержки лабораторного замера относительно телеметрии;
- синхронизацию `target` с телеметрией через `merge_asof`;
- генерацию лаговых и rolling-признаков;
- обучение и сравнение моделей Ridge, Random Forest и Gradient Boosting;
- временную валидацию `TimeSeriesSplit`;
- построение финального прогноза для тестового периода.

## Что делает ноутбук

### 1. Загрузка и подготовка данных
- читает телеметрию и лабораторные замеры;
- приводит временные колонки к `datetime`;
- сортирует данные и убирает дубликаты по времени.

### 2. EDA
- анализирует пропуски;
- строит графики телеметрии;
- показывает распределение и динамику `target`;
- оценивает интервалы между лабораторными замерами.

### 3. Подбор задержки и синхронизация
- перебирает задержки в диапазоне 10–15 минут;
- выбирает наиболее подходящую задержку по качеству согласования телеметрии и `target`;
- для каждого лабораторного замера берёт одно ближайшее прошлое состояние процесса;
- использует `merge_asof` с допуском `10min`.

### 4. Feature engineering
Создаются:
- исходные телеметрические признаки;
- лаги 5, 10, 15, 30 и 60 минут;
- скользящие статистики;
- разности и скорости изменения;
- календарные признаки.

### 5. Модели
Сравниваются:
- `Ridge`
- `RandomForestRegressor`
- `GradientBoostingRegressor`

Для моделей используется импутация пропусков. Для линейной модели добавляется масштабирование.

### 6. Оценка качества
Используются метрики:
- `MAE`
- `RMSE`
- `MAPE`
- `WAPE`
- `Directional Accuracy`

Дополнительно выполняются:
- анализ остатков;
- permutation importance;
- временная кросс-валидация `TimeSeriesSplit`.

### 7. Финальный прогноз
Лучшая модель обучается на всей train-выборке и формирует файл:

```text
predictions_test.csv
```
